# YouTube Channel Insight Analyzer — Project Start

## 1. 프로젝트 정의

이 프로젝트는 유튜브 카테고리를 선택하면 해당 분야의 주요 채널을 탐색하고, 잘되는 채널들의 성공 패턴을 분석하여 신규 채널 기획과 영상 제작 전략에 활용할 수 있는 웹 기반 인사이트 도구이다.

초기 사용자는 프로젝트 오너 1명이며, 개인용 내부 도구로 시작한다. 장기적으로는 유튜브 채널 기획자, 콘텐츠 전략가, 1인 크리에이터가 사용할 수 있는 분석 서비스로 확장 가능하다.

이 서비스의 핵심 목적은 단순한 채널 순위 조회가 아니라, 잘되는 채널의 에센스를 구조화하여 다음 의사결정에 활용하는 것이다.

- 어떤 카테고리에서 어떤 채널이 강한가?
- 해당 채널들은 왜 잘되는가?
- 제목, 썸네일, 영상 구성, 업로드 패턴에서 반복되는 성공 공식은 무엇인가?
- 신규 채널이 참고할 요소와 피해야 할 요소는 무엇인가?
- 어떤 세부 주제들이 관심을 끌 수 있을까? 특히 책, 철학, 문화, 심리, 자기계발, 커리어 분야에서.
- “잠들기전 교양이” 채널에 적용 가능한 전략은 무엇인가?

---

## 2. 프로젝트 배경

현재 기획 중인 유튜브 채널 “잠들기전 교양이”는 책, 철학, 문화, 심리, 자기계발, 커리어, IT 분야의 지식 콘텐츠를 기반으로 한다.

기존 책 기반 유튜브 채널은 단일 책 요약, 독서 리뷰, 자기계발 메시지, 지식 큐레이션 등으로 나뉘어 있다. 하지만 “잠들기전 교양이”는 영상 하나당 한 권의 책만 다루기보다, 여러 책과 관점을 하나의 주제로 엮어 차분하고 지적인 콘텐츠를 만드는 방향을 지향한다.

따라서 경쟁 채널과 레퍼런스 채널을 체계적으로 분석하고, 콘텐츠 기획에 바로 적용 가능한 전략 인사이트를 얻기 위한 전용 도구가 필요하다.

---

## 3. 제품 비전

사용자가 카테고리를 선택하면 해당 분야의 주요 유튜브 채널을 한눈에 비교하고, 각 채널의 성공 요인을 분석하며, 최종적으로 신규 영상 기획에 활용 가능한 전략을 얻는 도구를 만든다.

제품의 지향점은 다음과 같다.

- 유튜브 채널 리서치 자동화
- 채널 성공 패턴의 구조화
- 콘텐츠 전략 수립 지원
- 신규 영상 아이디어 생성 보조
- “잠들기전 교양이” 채널 운영 전략 도출

---

## 4. 핵심 사용자 시나리오

### Scenario 1. 카테고리별 상위 채널 탐색

사용자는 “책 / 독서” 카테고리를 선택한다. 서비스는 해당 분야의 주요 유튜브 채널 Top 10을 보여준다. 사용자는 구독자 수, 평균 조회수, 영상 수, 최근 업로드일을 비교한다.

### Scenario 2. 특정 채널의 성공 요인 분석

사용자는 특정 채널을 클릭한다. 서비스는 해당 채널의 포지셔닝, 콘텐츠 포맷, 제목 패턴, 썸네일 패턴, 업로드 패턴, 인기 영상 특징, 잘되는 이유를 요약한다.

### Scenario 3. 카테고리 공통 패턴 도출

사용자는 카테고리 내 Top 10 채널의 공통점을 확인한다. 서비스는 반복되는 성공 패턴, 과포화 포맷, 기회 영역, 신규 채널이 참고할 포인트를 제안한다.

### Scenario 4. “잠들기전 교양이” 적용 전략 확인

사용자는 분석 결과를 바탕으로 “잠들기전 교양이” 채널에 적용 가능한 콘텐츠 포맷, 제목 방향, 썸네일 방향, 영상 구성 방향을 확인한다.

### Scenario 5. 신규 영상 기획안 생성

사용자는 주제나 책 제목을 입력한다. 서비스는 분석된 레퍼런스 패턴을 바탕으로 영상 제목 후보, 구성안, 오프닝 훅, 썸네일 방향, 차별화 포인트를 제안한다.

## 5. 에이전트 운영 구조

이 프로젝트는 리서치, 기획, 디자인, 개발 에이전트가 협업하는 방식으로 진행한다.
각각의 에이전트 지침은 `/agents`에서 지속적으로 관리한다

### 5.1 Research Agent

역할: 유튜브 채널과 콘텐츠 트렌드를 조사하고 분석한다.

책임:

- 카테고리별 주요 채널 후보 조사
- 채널별 기본 정보 정리
- 인기 영상 패턴 분석
- 제목 / 썸네일 / 영상 길이 / 업로드 주기 분석
- 경쟁 채널 간 공통점과 차이점 도출
- “잠들기전 교양이”에 적용 가능한 요소 도출

산출물:

- 카테고리별 채널 리스트
- 채널별 분석 리포트
- 인기 영상 패턴 요약
- 경쟁 채널 비교표
- 참고할 요소 / 피해야 할 요소

분석 프레임:

1. What: 무엇을 하고 있는가?
2. Why: 왜 잘되는가?
3. Pattern: 반복되는 패턴은 무엇인가?
4. Gap: 비어 있는 기회는 무엇인가?
5. Apply: 우리 채널에 어떻게 적용할 수 있는가?

---

### 5.2 Planning Agent

역할: 서비스 기능, 사용자 흐름, 정보 구조, 분석 로직, MVP 범위를 정의한다.

책임:

- PRD 작성
- MVP 범위 정의
- 사용자 시나리오 설계
- IA 설계
- 기능 우선순위 설정
- 데이터 구조 정의
- 분석 결과 출력 포맷 설계
- 에이전트 간 작업 흐름 정의

우선순위 원칙:

1. 개인 사용 목적에 맞게 빠르게 사용할 수 있어야 한다.
2. 완벽한 자동화보다 인사이트 품질이 중요하다.
3. 초기에는 수동 입력과 mock data를 허용한다.
4. UI와 데이터 구조를 먼저 안정화한다.
5. 이후 YouTube API와 AI 분석 기능을 단계적으로 연결한다.

---

### 5.3 Design Agent

역할: 화면 구조, 정보 시각화, 분석 카드, 인사이트 보드 UI를 설계한다.

디자인 방향:

- DESIGN-airbnb.md 파일을 디자인 지침으로 삼고 사이트의 모든 컴포넌트와 UI를 디자인한다.
- 전략 분석 도구처럼 보여야 한다.
- 데이터 대시보드와 콘텐츠 기획 보드의 중간 성격을 가진다.
- 복잡한 정보를 쉽게 비교할 수 있어야 한다.
- 감성적 크리에이터 도구보다는 실용적인 인사이트 툴에 가깝다.
- “잠들기전 교양이”의 차분하고 지적인 분위기를 일부 반영한다.

UI 키워드:

- Modern
- Calm
- Strategic
- Analytical
- Minimal
- Card-based
- Readable density

---

### 5.4 Development Agent

역할: 웹 애플리케이션을 구현한다.

권장 기술 스택:

- Framework: Next.js
- Language: TypeScript
- Styling: Tailwind CSS
- UI Components: shadcn/ui
- Database: SQLite 또는 Supabase
- ORM: Prisma
- API: YouTube Data API v3
- AI Analysis: OpenAI API 또는 추후 교체 가능한 LLM 인터페이스
- Hosting: Vercel 

초기 원칙:

- MVP 1단계는 API 없이 mock data 기반으로 구현한다.
- 기능보다 구조와 흐름을 먼저 완성한다.
- 분석 프롬프트는 `/prompts` 폴더에 별도 관리한다.
- 문서는 `/docs` 폴더에 관리한다.

## 6. Codex 기본 작업 원칙

Codex는 다음 원칙에 따라 작업한다.

1. 한 번에 너무 많은 기능을 구현하지 않는다.
2. 먼저 MVP 1단계부터 구현한다.
3. 모든 주요 변경 전에는 간단한 작업 계획을 작성한다.
4. 작업 후 변경된 파일 목록과 역할을 요약한다.
5. UI는 mock data 기반으로 먼저 완성한다.
6. API 연동은 UI와 데이터 구조가 안정된 뒤 진행한다.
7. 분석 프롬프트는 별도 파일로 관리한다.
8. 서비스 목적은 단순 순위 조회가 아니라 전략 인사이트 도출이다.
9. 모든 화면은 “채널 운영 전략에 어떤 도움이 되는가?”를 기준으로 설계한다.
10. “잠들기전 교양이 적용 포인트”를 주요 분석 결과에 포함한다.

---

## 7. 최초 작업 요청

1. Next.js + TypeScript + Tailwind CSS 프로젝트 생성

3. `/docs`, `/prompts`, `/data` 폴더 생성
4. 기본 문서 파일 생성
5. 기본 프롬프트 파일 생성
6. mock data 생성
7. Dashboard, Category Explorer, Channel Detail, Insight Board, Strategy Generator, Settings 화면 생성
8. 공통 레이아웃과 네비게이션 생성
9. 실제 API 없이 mock data로 화면이 동작하도록 구현
