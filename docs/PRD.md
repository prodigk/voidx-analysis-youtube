# PRD — YouTube Channel Insight Analyzer

## 1. 제품명

YouTube Channel Insight Analyzer

가칭:

- Channel Essence Finder
- YouTube Strategy Board
- Content Pattern Analyzer
- 잠들기전 교양이 Research Board

---

## 2. 제품 목적

이 제품은 유튜브 카테고리별 주요 채널을 탐색하고, 성공 채널의 콘텐츠 전략을 분석하여 신규 채널 운영과 영상 기획에 활용 가능한 인사이트를 제공하는 웹 서비스이다.

초기 목적은 “잠들기전 교양이” 채널을 기획하고 운영하는 과정에서 참고할 레퍼런스 분석 도구를 만드는 것이다.

핵심 가치는 다음과 같다.

- 잘되는 채널의 패턴을 빠르게 파악한다.
- 채널별 차별화 포인트를 구조화한다.
- 콘텐츠 기획에 바로 쓸 수 있는 전략 인사이트를 얻는다.
- 신규 영상 아이디어를 분석 기반으로 만든다.

---

## 3. 문제 정의

유튜브 채널을 새로 기획하거나 운영할 때 레퍼런스 조사는 필수지만, 일반적인 방식은 비효율적이다.

현재 문제:

- 카테고리별 주요 채널을 찾는 데 시간이 오래 걸린다.
- 채널이 왜 잘되는지 감으로 판단하기 쉽다.
- 제목, 썸네일, 콘텐츠 포맷의 반복 패턴을 체계적으로 보기 어렵다.
- 레퍼런스 분석 결과가 실제 영상 기획으로 연결되지 않는 경우가 많다.
- 책 / 교양 / 자기계발 / IT처럼 여러 분야를 엮는 채널은 포지셔닝 참고 자료가 분산되어 있다.

---

## 4. 해결 방향

카테고리 선택 → Top 채널 확인 → 채널별 패턴 분석 → 카테고리 공통 인사이트 → 신규 채널 적용 전략 → 영상 기획안 생성으로 이어지는 분석 흐름을 제공한다.

## 5. 핵심 사용자

### Primary User

- 프로젝트 오너
- “잠들기전 교양이” 채널 기획자
- 책 기반 지식 콘텐츠를 만드는 개인 크리에이터

---

## 6. 핵심 기능 요구사항

### 6.1 카테고리 관리

사용자는 분석하고 싶은 유튜브 카테고리를 선택하고 관리할 수 있다.

초기 카테고리:

- 책 / 독서
- 인문학 / 교양
- 철학
- 심리학
- 자기계발
- 커리어
- IT / 테크
- 경제 / 비즈니스
- 지식 큐레이션

기능:

- 카테고리 목록 조회
- 카테고리 선택
- 카테고리 추가
- 카테고리 수정
- 카테고리 삭제

MVP 1단계에서는 mock data 기반으로 구현한다.

---

### 6.2 채널 리스트

카테고리를 선택하면 해당 카테고리의 주요 채널 목록을 보여준다.

표시 정보:

- 채널명
- 채널 URL
- 구독자 수
- 총 조회수
- 영상 수
- 최근 업로드일
- 대표 주제
- 채널 요약
- 분석 상태

정렬 기준:

- 구독자 수
- 평균 조회수
- 최근 업로드일
- 영상 수

MVP 1단계에서는 수동 입력된 mock channel data를 사용한다.

---

### 6.3 채널 상세 분석

채널을 선택하면 상세 분석 정보를 보여준다.

분석 항목:

1. 채널 포지셔닝
2. 타겟 시청자
3. 핵심 콘텐츠 주제
4. 콘텐츠 포맷
5. 제목 패턴
6. 썸네일 패턴
7. 업로드 패턴
8. 인기 영상 특징
9. 잘되는 이유
10. 차별화 포인트
11. 약점 또는 리스크
12. 신규 채널이 참고할 요소
13. “잠들기전 교양이” 적용 포인트

---

### 6.4 카테고리 인사이트

카테고리 내 여러 채널을 비교하여 공통 패턴과 기회 영역을 도출한다.

분석 항목:

- 공통 성공 패턴
- 자주 등장하는 제목 구조
- 썸네일 공통 패턴
- 평균 영상 길이
- 인기 콘텐츠 포맷
- 과포화된 포맷
- 비어 있는 기회 영역
- 신규 채널 진입 전략
- “잠들기전 교양이” 적용 전략

---

### 6.5 전략 생성기

사용자가 주제나 책 제목을 입력하면 분석 결과를 바탕으로 영상 기획안을 생성한다.

입력값:

- 카테고리
- 참고 채널
- 책 제목
- 주제
- 원하는 톤
- 타겟 시청자

출력값:

- 영상 콘셉트
- 제목 후보
- 썸네일 방향
- 오프닝 훅
- 영상 구성안
- 핵심 메시지
- 참고한 성공 패턴
- 차별화 포인트
- 주의할 리스크

MVP 1단계에서는 mock output 또는 수동 생성 구조로 구현한다.

## 7. 비기능 요구사항

### 7.1 사용성

- 개인용 도구이므로 복잡한 온보딩은 필요 없다.
- 좌측 네비게이션 또는 상단 네비게이션으로 주요 화면에 빠르게 접근할 수 있어야 한다.
- 분석 결과는 긴 보고서보다 카드와 섹션 구조로 읽기 쉬워야 한다.

### 7.2 확장성

- YouTube API 연동을 나중에 추가할 수 있어야 한다.
- AI 분석 모델을 교체할 수 있도록 분석 로직을 분리한다.
- 카테고리, 채널, 영상, 분석 결과 데이터 모델을 분리한다.

### 7.3 성능

- MVP 단계에서는 mock data 기반이므로 별도 성능 최적화는 최소화한다.
- 향후 API 연동 시 데이터 캐싱 구조를 고려한다.

### 7.4 보안

- 초기 개인용 도구에서는 복잡한 인증을 제외한다.
- API Key는 `.env`에서 관리한다.
- 추후 외부 공개 시 인증과 권한 관리를 추가한다.

## 9. 성공 기준

MVP 1 성공 기준:

- 주요 화면을 이동하며 전체 서비스 구조를 이해할 수 있다.
- mock data만으로도 채널 분석 도구의 사용 흐름이 보인다.
- 채널 상세와 인사이트 보드에서 전략적 판단을 할 수 있다.
- “잠들기전 교양이” 적용 포인트가 명확히 보인다.

제품 장기 성공 기준:

- 카테고리별 성공 채널을 빠르게 파악할 수 있다.
- 잘되는 채널의 패턴을 반복 가능하게 분석할 수 있다.
- 신규 영상 기획안의 품질이 향상된다.
- 레퍼런스 조사 시간이 줄어든다.
- 채널 운영 전략 수립에 직접 기여한다.
